```
/ - здесь лежат главные сущности (модели)

├─api         - описание api проекта в виде сваггер и postman файлов

├─build       - docker-файлы, нужные для билда проекта

├─cmd         - entrypoint проекта

├─config      - конфиг файлы, нужны только для локального запуска. На stage и prod

│               используется чтение из consul/vault

├─deployments - файлы для деплоя на aws, helm-charts, etc.

├─internal─┐  - мякотка проекта

│          ├─cache      - реализации кешей

│          ├─grpc       - gRPC-хэндлеры

│          ├─http       - http-хэндлеры (rest)

│          ├─repository - слой - реализация паттерна "репозиторий"

│          └─services   - сервисный слой приложения (бизнес-логика)

├─migrations  - sql-миграции, работает через make migrate (используется мигратор goose)

└─scripts     - всякие служебные скрипты
```

Всем привет, вот примерная структура проекта
1. cmd - в этой папке у вас будет лежать main - точка входа, обычно это так cmd/service-name/main.go
2. deployments - это папка для конфигурации деплоя приложения, вам она не нужна


——
stage и prod окружения скорей всего не нужны будут, достаточно локального запуска (под вопросом)

——
для админки можно будет сделать отдельный бинарник, то есть в cmd будет admin/main.go - cmd/admin/main.go
